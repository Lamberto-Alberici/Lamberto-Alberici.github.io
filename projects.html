Motherhood Wage Penalty Analysis

Stata Regression Analysis Statistical Modeling
ðŸ“„ View Full Paper

Overview

Investigated whether mothers in the U.S. face a wage penalty using regression analysis on microdata from the 2024 IPUMS CPS ASEC survey. Analyzed the relationship between number of children and wages while controlling for education, experience, hours worked, marital status, race, ethnicity, and occupation. Applied survey weights to account for complex sampling methods and ensure statistical accuracy.

Approach

  • Built multiple OLS regression models with log-transformed income as the dependent variable
  • Controlled for key variables: education years, weekly hours, work experience, marital status, race/ethnicity, and occupation
  • Applied ASEC survey weights to correct for sampling bias and improve model accuracy
  • Calculated robust standard errors and 95% confidence intervals to assess statistical significance
  • Analyzed wage effects across different income groups ($20K-$60K vs. >$60K) to examine how the penalty varies by earnings level

Key Findings

  • Initial models suggested a negative correlation between number of children and wages, but the effect lost statistical significance when proper survey weights were applied
  • The weighted model showed a 0.25% decrease in wages per child, but confidence intervals included zero, indicating no conclusive evidence of a motherhood wage penalty in 2024
  • Lower-income mothers ($20K-$60K) experienced a 0.76% wage decrease per child, while higher-income mothers (>$60K) saw a 1.13% wage increase, suggesting the penalty disproportionately affects lower-wage workers
  • Results contrast with previous studies from 2004-2013 that found significant penalties, potentially indicating improved workplace equity or methodological differences
  • Limitations: The analysis acknowledges potential omitted variable bias (e.g., motivation, career choices) and sampling selection bias. While this study found no statistically significant penalty, mothers may still face wage disadvantages that require different methodological approaches or state-level analysis to detect

Housing Price Prediction Using Machine Learning

Python Tableau Machine Learning Regression Analysis

Collaborated with Scott Ballentine and Connor Agler

Overview

Group project analyzing housing prices in Ames, Iowa using a dataset of 2,919 homes and 80 features. Applied multiple linear regression and recursive feature selection to identify key price drivers and analyzed how economic factors like the 2008 housing crisis and mortgage rates impact housing affordability.

Approach

  • Built two OLS regression models: one using the top 10 correlated variables, another using recursive feature selection across all 388 variables
  • Conducted data cleaning, handled missing values, and converted categorical variables to dummy variables
  • Performed three additional analyses examining the 2008 housing crisis impact, neighborhood-level resilience, and mortgage interest rate effects on affordability
  • Analyzed neighborhood trajectories to identify which areas appreciated or depreciated during the housing crisis

Key Findings

  • The optimized model achieved an adjusted RÂ² of 0.508, explaining 50.8% of price variability through 30 statistically significant predictors
  • Living area and overall quality emerged as the strongest predictors of housing prices
  • During the 2008 crisis, larger neighborhoods showed lower volatility in price changes compared to smaller neighborhoods
  • Higher-end neighborhoods demonstrated no statistical advantage in surviving the housing crisis compared to lower-end neighborhoods
  • Mortgage interest rates had a significant impact on affordability: a 1% increase in rates drives monthly payments up by 15-25%

Interactive Dodging Game

Java OOP Game Development
View Code on GitHub

Overview

Developed an interactive minigame where players control an avatar by dragging with the mouse to dodge incoming bombs. Features character sprites from the TV show Arcane. Built as a project for Principles of Software Development, demonstrating core Java programming concepts including event handling, collision detection, and real-time user input processing.

Approach

  • Implemented object-oriented design with separate classes for game entities (player avatar, bombs, game state)
  • Used Java event listeners to capture mouse movements and update player position in real-time
  • Developed collision detection algorithms to determine when bombs hit the player
  • Created a scoring system and difficulty progression as gameplay continues
  • Designed the user interface with Java graphics libraries

Key Features

  • Smooth, responsive mouse-drag controls for intuitive gameplay
  • Dynamic bomb spawning with increasing difficulty over time
  • Real-time collision detection and game state management
  • Clean, functional UI showing score and game status
